# Warpath Reels Audio

This directory is the production-facing audio asset location consumed by the frontend audio manifest.

## Current Assets

The current background music uses OpenGameArt orchestral tracks as a legally cleaner immediate upgrade:

- `oga_hope_orchestral_battle_music_bpm165.ogg` from OpenGameArt, used as the base loop.
- `oga_determined_pursuit_loop.wav` from OpenGameArt, used as the bonus loop.
- `oga_the_final_battle.ogg` is included as an alternate cinematic source candidate.

OpenGameArt source pages:

- Hope Orchestral Battle Music: https://opengameart.org/content/hope-orchestral-battle-music
- Determined Pursuit Epic Orchestra Loop: https://opengameart.org/content/determined-pursuit-epic-orchestra-loop
- The Final Battle: https://opengameart.org/content/the-final-battle

Treat these as `license-pending` for final StakeEngine upload until the exact track pages/license records are archived in the project legal/provenance folder.

The committed placeholder `.wav` SFX files are deterministic local integration placeholders generated by:

```bash
python3 tools/generate_audio_placeholders.py
```

They prove the StakeEngine-style frontend flow with static assets, manifest loading, and emitterEvent-driven playback. They are not final mastered SFX.

## Production Generation Target

Final music should be generated offline with ACE-Step or another commercially usable model, then mastered and exported into this same manifest shape.

ACE-Step references:

- `spin_start.wav`: spin launch.

- `feature_intro.wav`: Warpath Spins intro.
- `win_small.wav`, `win_big.wav`, `win_mega.wav`, `win_legendary.wav`: win tiers.
- `manifest.json`: file map and playback volumes.

## Frontend Integration Contract

The frontend does not infer or generate music at runtime. It loads `manifest.json`, decodes static audio buffers, then responds to typed emitterEvents from `bookEventHandlerMap` and UI events. This matches the StakeEngine frontend pattern of bookEvents -> emitterEvents -> component-level handlers.
